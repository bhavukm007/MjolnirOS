# Project Rules

These rules override all other documentation.

1. Read every document inside the docs folder before writing code.

2. MASTER_SPEC.md is the highest priority document.

3. Never redesign the architecture.

4. Never rename folders without approval.

5. Never delete working functionality.

6. Never skip phases.

7. Implement only the current phase.

8. Every phase must leave the project runnable.

9. Run tests before completing every phase.

10. Fix all errors before committing.

11. Never use paid APIs.

12. Use Ollama as the default AI provider.

13. Never replace local AI with cloud AI.

14. Always update README.md.

15. Always update CHANGELOG.md.

16. Use Conventional Commits.

17. Push to GitHub after every completed phase.

18. Stop after completing the assigned phase.

19. Ask for clarification instead of guessing.

20. Write production-quality code only.
