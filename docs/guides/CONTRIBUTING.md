# Contributing

Thank you for contributing to MjolnirOS.

---

## Workflow

1. Fork the repository.
2. Create a feature branch.
3. Implement the changes.
4. Run the tests.
5. Update the documentation.
6. Create a pull request.

---

## Commit Format

Use Conventional Commits.

Examples:

```text
feat(memory): add semantic search

fix(browser): resolve login issue

docs: update roadmap
```

---

## Coding Rules

- Follow [MASTER_SPEC.md](../reference/MASTER_SPEC.md)
- Follow [ARCHITECTURE.md](../reference/ARCHITECTURE.md)
- Follow [CODING_GUIDELINES.md](CODING_GUIDELINES.md)
- Never commit secrets.
- Never leave TODOs.
- Add tests for new features.

---

## Pull Requests

Pull requests must include:

- Description
- Testing
- Documentation updates

---

## Branches

- `main`
- `feature/*`
- `fix/*`
- `refactor/*`
