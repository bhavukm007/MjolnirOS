# Contributing

Thank you for contributing to MjolnirOS.

---

## Workflow

1. Fork repository
2. Create feature branch
3. Implement changes
4. Run tests
5. Update documentation
6. Create pull request

---

## Commit Format

Conventional Commits

Examples

feat(memory): add semantic search

fix(browser): resolve login issue

docs: update roadmap

---

## Coding Rules

- Follow MASTER_SPEC.md
- Follow ARCHITECTURE.md
- Follow CODING_GUIDELINES.md
- Never commit secrets
- Never leave TODOs
- Add tests for new features

---

## Pull Requests

Must include

- Description
- Testing
- Documentation updates

---
