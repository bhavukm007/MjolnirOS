# Contributing to MjolnirOS

Thank you for considering a contribution to MjolnirOS.

The contribution workflow, commit conventions, coding requirements, branch guidance, and pull-request expectations are maintained in the [contribution guide](docs/guides/CONTRIBUTING.md).

Before opening a pull request:

1. Search existing issues and pull requests for related work.
2. Keep the change focused and preserve the local-first security model.
3. Run the relevant tests and checks.
4. Update documentation when user-facing behavior or interfaces change.
5. Complete the pull-request template.

By participating, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
